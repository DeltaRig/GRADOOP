87     * is attached to the resulting vertices.

89     * Note, that this method used no statistics about the data graph which may result in bad runtime

91     *

93     * @return graph collection containing matching subgraphs
94     */

96      return query(query, new GraphStatistics(1, 1, 1, 1));

99    /**
100    * Evaluates the given query using the Cypher query engine. The engine uses default morphism strategies,
101    * which is vertex homomorphism and edge isomorphism. The vertex and edge data of the data graph elements
102    * is attached to the resulting vertices.
103    * <p>
104    * Note, that this method used no statistics about the data graph which may result in bad runtime
105    * performance. Use {@link #query(String, String, GraphStatistics)} to provide statistics for the query planner.
106    * <p>
107    * In addition, the operator can be supplied with a construction pattern allowing the creation of new graph
108    * elements based on variable bindings of the match pattern. Consider the following example:
109    * <br>
110    * {@code graph.query(
111    *  "MATCH (a:Author)-[:WROTE]->(:Paper)<-[:WROTE]-(b:Author) WHERE a <> b",
112    *  "(a)-[:CO_AUTHOR]->(b)")}
113    * <p>
114    * The query pattern is looking for pairs of authors that worked on the same paper.
115    * The construction pattern defines a new edge of type CO_AUTHOR between the two entities.
116    *
117    * @param query               Cypher query string
118    * @param constructionPattern Construction pattern
119    * @return graph collection containing the output of the construct pattern
120    */
121   default GC query(String query, String constructionPattern) {
122     return query(query, constructionPattern, new GraphStatistics(1, 1, 1, 1));
123   }

133    */

247   /**
248    * Creates a condensed version of the base graph by grouping vertices and edges based on given
249    * property keys.
250    * <p>
251    * Vertices are grouped by the given property keys. Edges are implicitly grouped along with their
252    * incident vertices and explicitly by the specified edge grouping keys. Furthermore, one can
253    * specify sets of vertex and edge aggregate functions which are applied on vertices/edges
254    * represented by the same super vertex/edge.
255    * <p>
256    * One needs to at least specify a list of vertex grouping keys. Any other argument may be
257    * {@code null}.
258    * <p>
259    * Note: To group vertices/edges by their type label, one needs to add the specific symbol
260    * {@link Grouping#LABEL_SYMBOL} to the respective grouping keys.
261    *
262    * @param vertexGroupingKeys       property keys to group vertices
263    * @param vertexAggregateFunctions aggregate functions to apply on super vertices
264    * @param edgeGroupingKeys         property keys to group edges
265    * @param edgeAggregateFunctions   aggregate functions to apply on super edges
266    * @param groupingStrategy         execution strategy for vertex grouping
267    * @return summary graph
268    * @see Grouping
269    */
270   default LG groupBy(
271     List<String> vertexGroupingKeys, List<AggregateFunction> vertexAggregateFunctions,
272     List<String> edgeGroupingKeys, List<AggregateFunction> edgeAggregateFunctions,
273     GroupingStrategy groupingStrategy) {
274 
275     Objects.requireNonNull(vertexGroupingKeys, "missing vertex grouping key(s)");
276     Objects.requireNonNull(groupingStrategy, "missing vertex grouping strategy");
277 
278     Grouping.GroupingBuilder builder = new Grouping.GroupingBuilder();
279 
280     builder.addVertexGroupingKeys(vertexGroupingKeys);
281     builder.setStrategy(groupingStrategy);
282 
283     if (edgeGroupingKeys != null) {
284       builder.addEdgeGroupingKeys(edgeGroupingKeys);
285     }
286     if (vertexAggregateFunctions != null) {
287       vertexAggregateFunctions.forEach(builder::addVertexAggregateFunction);
288     }
289     if (edgeAggregateFunctions != null) {
290       edgeAggregateFunctions.forEach(builder::addEdgeAggregateFunction);
291     }
292     return callForGraph(builder.build());
293   }
